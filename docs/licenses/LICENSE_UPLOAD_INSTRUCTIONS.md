# ScyllaDB License Upload Instructions

## Overview

ScyllaDB now supports commercial licensing through signed license files. Licenses are cryptographically signed using Ed25519 to prevent tampering and are managed via a REST API.

## Prerequisites

1. **Running ScyllaDB instance** with license API enabled
2. **Valid license file** (generated by ScyllaDB licensing team)
3. **curl** or similar HTTP client for API access

## Generating License Files (Internal Use Only)

### Generate Master Keypair (One-Time Setup)

```bash
cd /home/piotrsmaron/src/scylla_sa_license
./tools/scylla-license-gen.py generate-keypair > keypair.txt
```

**⚠️ Security Warning:** Store the seed from `keypair.txt` in a secure offline location. This seed is required to generate all customer licenses.

### Generate Customer License

```bash
# Unlimited license (no vCPU/storage limits, never expires)
./tools/scylla-license-gen.py generate-license \
  --seed <hex_seed_from_keypair> \
  --customer "AcmeEnterprise" \
  --unlimited \
  --output acme_enterprise.key

# Limited license with expiry
./tools/scylla-license-gen.py generate-license \
  --seed <hex_seed_from_keypair> \
  --customer "StartupCo" \
  --vcpus 128 \
  --storage-tb 10 \
  --expiry "2026-12-31" \
  --output startup_co.key

# Verify license before sending to customer
./tools/scylla-license-gen.py verify-license \
  --seed <hex_seed_from_keypair> \
  --license-file startup_co.key
```

## Uploading Licenses via API

### Check Current License Status

```bash
curl http://localhost:10000/v2/license/status | jq
```

**Response (no license):**
```json
{
  "status": "no_license",
  "message": "No license installed"
}
```

**Response (valid license):**
```json
{
  "status": "valid",
  "customer_id": "AcmeEnterprise"
}
```

**Response (expired license):**
```json
{
  "status": "expired",
  "customer_id": "StartupCo",
  "message": "License expired on 2025-12-31"
}
```

### Upload New License

```bash
curl -X POST http://localhost:10000/v2/license/upload \
  -H "Content-Type: text/plain" \
  --data-binary @license.key
```

**Success Response:** Empty response with HTTP 200

**Error Responses:**
- `{"message": "Invalid license signature", "code": 400}` - License tampered or wrong public key
- `{"message": "Invalid license format...", "code": 400}` - Malformed license file
- `{"message": "License expired...", "code": 400}` - License expiry date has passed

### Check License Usage

```bash
curl http://localhost:10000/v2/license/usage | jq
```

**Response (unlimited license):**
```json
{
  "customer_id": "AcmeEnterprise",
  "expiry_timestamp": 0,
  "current_vcpus": 16,
  "current_storage_bytes": 5497558138880,
  "vcpu_limit_exceeded": false,
  "storage_limit_exceeded": false
}
```

**Response (limited license):**
```json
{
  "customer_id": "StartupCo",
  "expiry_timestamp": 1798675200,
  "max_vcpus": 128,
  "max_storage_bytes": 10995116277760,
  "current_vcpus": 16,
  "current_storage_bytes": 5497558138880,
  "vcpu_limit_exceeded": false,
  "storage_limit_exceeded": false
}
```

**Field Descriptions:**
- `expiry_timestamp`: Unix timestamp (ms), 0 = never expires
- `max_vcpus`: Maximum vCPUs allowed, null = unlimited
- `max_storage_bytes`: Maximum storage in bytes, null = unlimited
- `current_vcpus`: Current vCPU count across cluster
- `current_storage_bytes`: Current storage usage in bytes
- `vcpu_limit_exceeded`: True if current usage exceeds license
- `storage_limit_exceeded`: True if storage exceeds license

### Delete License (Testing/Admin)

```bash
curl -X DELETE http://localhost:10000/v2/license
```

## Multi-Node Clusters

Licenses are automatically replicated via Raft across all nodes in the cluster.

**Upload to any node:**
```bash
# Upload to first node - automatically replicates to all nodes
curl -X POST http://node1:10000/v2/license/upload \
  -H "Content-Type: text/plain" \
  --data-binary @license.key

# Wait ~1 second for Raft replication, then verify on other nodes
curl http://node2:10000/v2/license/status | jq
curl http://node3:10000/v2/license/status | jq
```

All nodes will show the same license status after replication.

## License File Format

License files contain exactly 2 lines:

```
SCYLLA_LICENSE:v1:<customer_id>:<expiry_timestamp>:<max_vcpus>:<max_storage_tb>
<128_char_hex_ed25519_signature>
```

**Example:**
```
SCYLLA_LICENSE:v1:AcmeEnterprise:0:4294967295:0
6eed593e8255f735cd4553cbb327dd28e6b8b58266d085b118127bb70925e2c5...
```

**Field Values:**
- `<customer_id>`: Company/customer identifier (max 64 chars)
- `<expiry_timestamp>`: Unix timestamp (seconds), 0 = never expires
- `<max_vcpus>`: 4294967295 = unlimited, otherwise max vCPU count
- `<max_storage_tb>`: 0 = unlimited, otherwise max storage in TB
- `<signature>`: 128-char hex-encoded Ed25519 signature

## Verification from Database

Licenses are stored in the `system.licenses` Raft table:

```bash
cqlsh -e "SELECT key, customer_id, expiry_timestamp FROM system.licenses;"
```

**Note:** The licenses table is **NOT accessible via regular CQL** operations (INSERT/UPDATE/DELETE). Only the REST API can modify licenses.

## Security Features

1. **Ed25519 Signatures** - Cryptographically prevents license tampering
2. **Embedded Public Key** - Verification key is obfuscated in ScyllaDB binary
3. **Private Key Isolation** - Only ScyllaDB licensing team has the private key
4. **Raft Replication** - Ensures consistency across cluster nodes
5. **CQL Protection** - Licenses table not writable via CQL, only REST API
6. **Grace Period Protection** - Grace period timestamps are also cryptographically signed

## Grace Period Behavior

When a license expires:

1. **Day 0-7**: Grace period active
   - All operations continue normally
   - Warning logs generated
   - Status API shows grace period status

2. **Day 8+**: Grace period expired
   - **Writes are blocked** cluster-wide
   - Reads continue to work
   - Administrators must reduce usage or upload new license

## Troubleshooting

### Invalid Signature Error

```bash
# Verify license with tool
./tools/scylla-license-gen.py verify-license \
  --seed <seed> \
  --license-file license.key
```

**Common causes:**
- License file corrupted during transfer
- Wrong public key embedded in ScyllaDB binary
- License tampered with

### License Expired Error

Check expiry date:
```bash
./tools/scylla-license-gen.py verify-license \
  --seed <seed> \
  --license-file license.key
```

Generate new license with extended expiry.

### Connection Refused

Verify ScyllaDB is running and API is accessible:
```bash
curl http://localhost:10000/v2/license/status
```

Check logs:
```bash
tail -f /var/log/scylla/scylla.log | grep -i license
```

### License Not Persisting After Restart

Check Raft replication is working:
```bash
# Should show licenses table
cqlsh -e "DESCRIBE TABLE system.licenses;"
```

Verify at least quorum of nodes are online.

## Example Workflow: Renewing License

```bash
# 1. Check current license
curl http://localhost:10000/v2/license/usage | jq

# 2. Generate new license with extended expiry
./tools/scylla-license-gen.py generate-license \
  --seed $SEED \
  --customer "ExistingCustomer" \
  --vcpus 256 \
  --storage-tb 20 \
  --expiry "2027-12-31" \
  --output renewed_license.key

# 3. Upload new license (replaces old one)
curl -X POST http://localhost:10000/v2/license/upload \
  -H "Content-Type: text/plain" \
  --data-binary @renewed_license.key

# 4. Verify new license is active
curl http://localhost:10000/v2/license/status | jq
curl http://localhost:10000/v2/license/usage | jq
```
